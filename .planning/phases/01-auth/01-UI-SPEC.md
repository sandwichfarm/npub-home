# Phase 1 — UI Design Contract: Auth

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind v4 with CSS custom properties in app.css) |
| Preset | not applicable |
| Component library | none (hand-rolled Svelte components) |
| Icon library | none (text/arrow characters used in existing code) |
| Font | system-ui (browser default, no web font loaded) |

**Note:** This is a Svelte 5/SvelteKit project. shadcn/ui is React-only and is not applicable. The design system is the existing Tailwind v4 token layer defined in `src/app.css`. All new components must consume the same CSS custom property tokens (`--color-background`, `--color-primary`, etc.) as established components.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tab indicator underlines, inline badge padding |
| sm | 8px | Input label gap, button internal padding (vertical) |
| md | 16px | Modal horizontal padding, input field padding, default gap between form elements |
| lg | 24px | Modal vertical padding (top/bottom), section spacing inside modal |
| xl | 32px | Modal width constraint padding (outside) |
| 2xl | 48px | N/A in Phase 1 |
| 3xl | 64px | N/A in Phase 1 |

Exceptions:
- Touch targets for the tab buttons must be minimum 44px tall (accessibility minimum for interactive controls).
- The footer "Login" / "Logout" link uses 8px vertical padding to match existing footer text rhythm.

## Color

All values are Tailwind CSS custom property tokens from `src/app.css`. Do not hard-code hex values.

| Role | CSS Token | Approximate HSL | Usage |
|------|-----------|-----------------|-------|
| Dominant (60%) | `bg-background` | hsl(228 20% 10%) | Modal backdrop overlay, page background |
| Secondary (30%) | `bg-card` / `bg-secondary` | hsl(228 20% 12%) / hsl(228 20% 18%) | Modal panel background (card), tab bar background (secondary), input background |
| Accent (10%) | `text-primary` / `border-primary` | hsl(258 70% 60%) — purple | Active tab indicator underline, focused input ring, QR code accent, owner badge dot |
| Destructive | `text-destructive` | hsl(0 72% 51%) — red | Error messages, connection failure state only |

Accent reserved for:
1. Active tab indicator (underline on the selected "Extension" or "Remote Signer" tab)
2. Input focus ring on the relay URL field
3. The loading spinner border-t color during NIP-46 connection
4. The owner badge dot next to "Logout" in the footer
5. QR code foreground color (lean-qr output)

**Do not** use accent on the modal close button, passive tab labels, or footer text.

## Interaction States

### Login Modal — NIP-07 Tab

| State | Visual |
|-------|--------|
| Extension detected | "Connect Extension" button enabled, `bg-primary text-primary-foreground`, hover opacity 80 |
| Extension not detected | "Connect Extension" button disabled (`opacity-50 cursor-not-allowed`), tooltip "No extension detected" on hover |
| Connecting | Button replaced by spinner (`LoadingSpinner` with `message="Connecting..."`) |
| Error | `ErrorMessage` component below button with error text in `text-destructive` |

### Login Modal — NIP-46 Tab

| State | Visual |
|-------|--------|
| Idle | QR code displayed, relay field pre-filled with `wss://bucket.coracle.social`, paste input empty |
| Relay field edited | QR code and URI update live on `input` event (no debounce required — lean-qr is synchronous) |
| URI pasted | Relay field updates to show the relay extracted from pasted URI; QR updates |
| Connecting | Spinner replaces QR area, "Waiting for approval..." label |
| Error | `ErrorMessage` below paste field |

### Footer Area

| State | Visual |
|-------|--------|
| Logged out | Small `text-sm text-muted-foreground` text link "Login" |
| Logged in as owner | "Logout" link (`text-sm text-muted-foreground`) + small purple dot badge (`w-2 h-2 rounded-full bg-primary inline-block ml-1`) |
| Not owner (logged in as different key) | Footer unchanged — shows "Login" (the user is not this page's owner; login link remains for the actual owner) |

### Owner Indicator (page body, Phase 1 placeholder)

| State | Visual |
|-------|--------|
| Owner confirmed | Small bar above nsite list: `text-sm font-semibold text-primary` reading "Logged in as owner" with a logout button (`text-xs text-muted-foreground underline`) inline |

## Accessibility

- Modal must trap focus when open (use Svelte's `use:focusTrap` action or manual tabindex management).
- Modal must be closeable via `Escape` key.
- Tab buttons must have `role="tab"` and `aria-selected` attributes.
- Disabled "Connect Extension" button must have `aria-disabled="true"` and the tooltip text in `title` attribute.
- QR code image must have `alt="NIP-46 connection QR code"`.
- Owner badge dot must have `aria-label="Owner"` or be wrapped in a `<span>` with screen-reader text.
