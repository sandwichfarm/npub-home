## Color

All values map to existing CSS custom properties. No new color tokens are introduced in this phase.

| Role | CSS Token | Tailwind Class | Usage |
|------|-----------|----------------|-------|
| Dominant (60%) | `--background` `228 20% 10%` | `bg-background` | Modal backdrop target surface |
| Secondary (30%) | `--card` `228 20% 12%` | `bg-card` | Modal panel background, theme cards |
| Muted surface | `--secondary` `228 20% 18%` | `bg-secondary` | Inputs, hover state on theme cards |
| Accent (10%) | `--primary` `258 70% 60%` | `text-primary`, `bg-primary`, `border-primary` | Selected theme card border, "Apply Theme" button fill, active swatch ring, nevent fetch retry link |
| Muted text | `--muted-foreground` | `text-muted-foreground` | Input labels, hint text, secondary copy, unselected theme names |
| Foreground | `--foreground` `210 40% 98%` | `text-foreground` | Theme card names (selected), modal title |
| Border | `--border` | `border-border` | Card borders, input borders, modal panel border |
| Destructive | `--destructive` `0 72% 51%` | `text-destructive` | Error message text under nevent input only |

Accent reserved for:
1. Selected theme card outline (`ring-2 ring-primary` or `border-primary`)
2. "Apply Theme" primary button background (`bg-primary text-primary-foreground`)
3. Selected swatch indicator ring
4. Inline retry link after nevent fetch failure (`text-primary underline`)
5. Active tab underline if tabs are used (matches LoginModal tab pattern)

Accent is NOT used for: unselected theme cards, general hover states, loading indicators, metadata text.

**Source:** `src/app.css` token definitions, existing component color usage patterns.

## Visual Hierarchy

**Primary focal point:** The 2-column theme card grid (`grid grid-cols-2 gap-3`) is the primary visual anchor of the modal. It occupies the majority of the modal's vertical space and is the first interactive region the user's eye should land on after the modal title. All other regions (nevent input, action row) are secondary and positioned below the grid.

Layout priority order within the modal:
1. Modal title + close button (header — orientation)
2. Theme card grid (primary focal point — selection)
3. nevent paste input (secondary — power user escape hatch)
4. Action row (tertiary — commit or dismiss)

### ThemePicker Layout

Two-panel modal structure:

**Panel:** `max-w-md w-full` (wider than LoginModal's `max-w-sm` to accommodate theme grid), `rounded-xl border border-border bg-card p-6 shadow-xl`. Follows existing modal pattern exactly.

**Backdrop:** `fixed inset-0 z-50 flex items-center justify-center bg-background/80 backdrop-blur-sm` — identical to LoginModal backdrop.

**Keyboard:** Escape closes and reverts preview (calls `clearTheme()` then reapplies active theme).

**Internal structure:**
1. Header row: modal title "Pick a Theme" (`text-xl font-semibold text-foreground`) + close button (`&#x2715;`, `text-muted-foreground hover:text-foreground`)
2. Theme grid: `grid grid-cols-2 gap-3` — 2-column grid of theme cards (primary focal point)
3. Divider
4. nevent paste input row
5. Action row: "Apply Theme" button (primary) + "Keep Current Theme" link-button (muted)

### Theme Card

Each theme card is a button (`role="button"` or `<button>`) with:
- Background: `bg-card rounded-lg border border-border p-3`
- Selected state: `border-primary ring-2 ring-primary`
- Hover state: `hover:border-primary/50 hover:bg-secondary`
- Content: 3 color circles (20px diameter, `rounded-full`) side by side for bg / text / primary colors, then theme name below as `text-sm text-foreground font-medium`
- Color circles are purely decorative (`aria-hidden="true"`)
- Keyboard: focusable, `Enter`/`Space` selects and previews

### Color Swatch Strip

3 circles in a `flex gap-2 mb-2` row:
- Circle 1: background color extracted from kind 36767 `c` tag
- Circle 2: text color
- Circle 3: primary color
- Each circle: `h-5 w-5 rounded-full` with `style="background-color: {hex}"` (raw hex from event tag, no conversion needed for display)
- No border needed — circles are on a `bg-card` surface so contrast is implicit

### nevent Paste Input

Label: `text-xs text-muted-foreground` reading "Add custom theme"
Input: `w-full rounded-md border border-border bg-secondary px-3 py-2 text-sm text-foreground placeholder-muted-foreground focus:outline-none focus:ring-2 focus:ring-primary` — identical to RemoteSignerTab input styling
Placeholder: `nevent1...`
Button: `mt-1 w-full rounded-lg bg-primary px-4 py-2 text-sm font-medium text-primary-foreground hover:opacity-80 disabled:opacity-50` labeled "Add Theme" — same as existing primary button pattern
Error: `text-sm text-destructive` inline below input, with a `text-primary underline` "Retry" link appended inline

### Loading State (Fetching Curated Themes)

Use existing `LoadingSpinner.svelte` with `message="Loading themes..."`. Shown in the theme grid area while fetching. Matches existing `py-16` vertical spacing.

### Loading State (Fetching nevent)

Inline: replace the "Add Theme" button label with "Fetching..." and `disabled={true}`. No spinner — too small a region.

### Success State (After Apply)

Modal auto-closes. No toast or banner. The live page already reflects the applied theme (via `applyTheme()` called on selection), so visual feedback is instant. The close itself is the confirmation.

### "Open Theme Picker" Button in OwnerBadge

Add to `OwnerBadge.svelte` alongside the existing Log out button:
```
class="text-xs font-normal text-muted-foreground underline hover:text-foreground"
```
Label: "Theme" (matching brevity of existing "Log out" link)

## Interaction States

| State | Trigger | Visual |
|-------|---------|--------|
| Default (no selection) | Modal opens | All cards unselected; page shows active theme |
| Card hover | Mouse/keyboard focus on card | `hover:bg-secondary hover:border-primary/50` |
| Card selected (preview) | Click/Enter on card | `border-primary ring-2 ring-primary`; `applyTheme()` called immediately |
| Loading curated themes | Initial mount | `LoadingSpinner` in grid area |
| nevent input loading | "Add Theme" clicked | Button disabled + label "Fetching..." |
| nevent error | Fetch failed | `text-destructive` error + "Retry" link below input |
| Apply in progress | "Apply Theme" clicked | Button disabled + label "Applying..." |
| Keep Current Theme | "Keep Current Theme" clicked or Escape | `clearTheme()` + reapply previously active theme; modal closes |
| Close (X button) | X clicked | Same as Keep Current Theme |
| Backdrop click | Click outside modal | Same as Keep Current Theme |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Modal title | Pick a Theme |
| Open picker button (in OwnerBadge) | Theme |
| Primary CTA | Apply Theme |
| Cancel action | Keep Current Theme |
| Close button aria-label | Close |
| nevent input label | Add custom theme |
| nevent input placeholder | nevent1... |
| nevent add button | Add Theme |
| nevent add button (loading) | Fetching... |
| Apply button (loading) | Applying... |
| Loading state message (grid) | Loading themes... |
| Empty state heading | No themes found |
| Empty state body | Could not load themes from relays. Check your connection and refresh. |
| nevent fetch error | Could not fetch theme. Check the nevent reference and try again. |
| nevent fetch error retry | Retry |
| Apply publish error | Failed to publish theme. Check your connection and try again. |
| Curated theme fallback name | Unnamed Theme |

No destructive actions in this phase. Apply is reversible (user can reopen picker and select a different theme). No confirmation dialog required (confirmed in CONTEXT.md).

"Keep Current Theme" is used instead of "Cancel" to make the revert action explicit — the label communicates what will happen (the previously active theme remains), not merely that the action is dismissed.

**Source:** CONTEXT.md — "Single 'Apply Theme' button — no confirmation dialog (non-destructive, easily undone)"; "Failed nevent fetch shows inline error under paste input: 'Could not fetch theme event' with retry option".

## Accessibility

- Modal: `role="dialog" aria-modal="true" aria-label="Pick a Theme" tabindex="-1"` — identical to LoginModal pattern

- Loading spinner: existing `LoadingSpinner` component (no aria changes needed)
- Focus trap: on modal open, focus the modal panel div (matches LoginModal: `tabindex="-1"`)
- Escape key handler on `<svelte:window>` — matches LoginModal pattern
